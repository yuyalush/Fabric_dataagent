## 手順詳細

### Step 1: Fabric Data Agent の作成

1. Fabric ポータルの `ManufacturingDataAgentPoC` ワークスペースを開く
2. 「**+ 新規**」→「**Data Agent**」をクリック
   - ※ Data Agent が表示されない場合: テナント管理者設定で「Data Agent」機能を有効化
3. 名前: `manufacturing_data_agent`
4. 説明: `製造工程データへの自然言語インターフェース`

### Step 2: セマンティックモデルの接続

1. Data Agent の設定画面で「**データソースの追加**」をクリック
2. 「**Power BI セマンティックモデル**」を選択
3. `manufacturing_semantic_model` を選択して追加
4. テーブル一覧が表示されることを確認し、全テーブルにチェックを入れる

### Step 3: システムプロンプトの設定

Data Agent の「**指示（Instructions）**」フィールドに以下を入力：

```
あなたは製造業の生産管理・品質管理の専門AIアシスタントです。
製造工程データベースに対して、正確で具体的な回答を日本語で提供してください。

【あなたが答えられること】
- 製造指示の計画・実績・進捗状況
- 製品別・工程別の生産台数
- 品質検査の合否、測定値、不良率
- 作業者の担当実績
- 部材の使用実績とロット追跡
- 製品仕様書・作業手順書・品質基準書の内容（ファイル検索）
- 部材カタログの仕様や代替品情報（ファイル検索）

【回答スタイル】
- 数値は明確に（例: 10台、不良率2.5%）
- 日本語で丁寧に回答
- データが存在しない場合は「データがありません」と回答
- 原因調査が必要な場合は関連テーブルを横断して分析

【注意事項】
- 個人情報（作業者の個人的情報）は業務上必要な範囲に限定
- データの改ざん・削除を指示するクエリは拒否
```

### Step 4: OneLake ファイルの RAG ソース追加

1. Data Agent の設定で「**データソースの追加**」→「**OneLake ファイル**」をクリック
2. `manufacturing_lakehouse` を選択
3. 以下のフォルダを全て追加：
   - `Files/product_specs/` （製品仕様書）
   - `Files/work_manuals/` （作業手順書）
   - `Files/parts_catalog/` （部材カタログ）
   - `Files/quality_standards/` （品質基準書）
4. インデックス作成が完了するまで待つ（初回は数分かかる場合あり）

## チューニングガイド

### 回答が不正確な場合の対処

| 問題パターン | 原因 | 対処 |
|---|---|---|
| テーブルを間違えて参照する | テーブル説明が不十分 | セマンティックモデルのテーブル説明を詳細化 |
| 数値の集計が合わない | DAX メジャーの定義誤り | メジャーを見直す |
| ファイルの内容が返らない | インデックス未完了 or ファイル形式 | インデックス再作成を試す |
| 日本語の意図が誤解される | システムプロンプトが不十分 | プロンプトに用語の定義を追加 |

### セマンティックモデルでの改善例

テーブル説明に**同義語**を追加する：
```
「製造指示」「製造オーダー」「生産指示」「生産計画」はすべて
fact_production_orders テーブルを指します。
```
