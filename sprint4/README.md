## 手順詳細

### Step 1: Data Agent を M365 Copilot に公開

Fabric Data Agent を M365 Copilot から利用できるようにするには、以下の設定が必要です。

#### 1-a. Fabric テナント設定の確認

1. Microsoft Fabric 管理ポータル（`app.fabric.microsoft.com/admin`）にアクセス
2. 「**テナント設定**」→「**Copilot と Azure OpenAI Service**」セクション
3. 「**Fabric に Copilot を使用する**」が有効になっていることを確認
4. 「**Data Agent を Teams に公開する**」が有効になっていることを確認

#### 1-b. Data Agent の公開

1. Fabric ポータルで `manufacturing_data_agent` を開く
2. 「**公開**」→「**Microsoft Teams に公開**」をクリック
3. 対象ユーザー/グループを設定して「**公開**」を確認

#### 1-c. M365 Copilot での確認

1. Microsoft Teams を開く
2. Copilot アプリ（Teams の左サイドバー）を選択
3. 「**エージェント**」→ `manufacturing_data_agent` を選択
4. 質問を入力して回答を確認

### Step 2: Copilot Studio でのカスタムエージェント作成（オプション）

より対話的な製造現場向けボットを作るため、Copilot Studio でカスタムエージェントを作成します。

#### 2-a. Copilot Studio でのエージェント作成

1. [Copilot Studio](https://copilotstudio.microsoft.com) にアクセス
2. 「**+ 新しいエージェントを作成**」をクリック
3. 名前: `製造支援エージェント`
4. 説明: `製造工程データへの質問と作業手順の案内を行うエージェントです`

#### 2-b. Data Agent の接続（Knowledge Source）

1. 作成したエージェントの「**ナレッジ**」タブを開く
2. 「**ナレッジの追加**」→「**Fabric Data Agent**」または「**Microsoft Copilot Studio コネクター**」
3. `manufacturing_data_agent` を選択して接続

#### 2-c. カスタムトピックの設定

**トピック 1: 生産実績の確認**
```
トリガーフレーズ:
- 製造実績を教えて
- 今日の生産状況は
- 何台完成した

アクション:
→ Data Agent に「[日付]の製造完了台数を製品別に教えて」を転送
```

**トピック 2: 品質問題の調査ガイド**
```
トリガーフレーズ:
- 不良品が出た
- 品質問題を調べたい
- 検査に失敗した

アクション:
→ 製品名・工程名を確認するための質問を返す
→ 入力を受け取り Data Agent に詳細クエリを転送
```

**トピック 3: 作業手順の確認**
```
トリガーフレーズ:
- 手順を教えて
- どうやって作業する
- SOPを見たい

アクション:
→ 製品名・工程名を確認
→ Data Agent に「[製品]の[工程]の手順を教えて」を転送
```

### Step 3: エンドユーザーテスト

#### テスト対象ロールと質問例

**製造担当（作業者）**
- 「今日の基板実装工程の作業手順を教えて」
- 「IGBTモジュールの取り付けで注意することは？」
- 「PART-001 の代替品はある？仕入先と単価は？」

**品質担当**
- 「今週の機能検査の不合格件数と内容を教えて」
- 「IVC-3000 の出力電流の検査基準値は？」
- 「先月の不良が多かった工程はどこ？」

**製造部長（管理職）**
- 「今月の生産ラインの稼働状況をサマリーして」
- 「2026年2月の製品別不良率を教えて」
- 「遅延が発生している製造指示はある？」

| 応答速度 | / 5 | |

### Step 4: セキュリティ確認

#### 行レベルセキュリティ（RLS）のテスト

PoC 段階ではロール分離は必須ではありませんが、本番を見越して確認します。

```sql
-- Power BI セマンティックモデルで RLS ロールを定義する例
-- 例: 製造担当は自部署のデータのみ参照可能

-- ロール: 製造1課
-- フィルター: dim_workers[department] = "製造1課"

-- Data Agent からのクエリは接続ユーザーの RLS が適用される
-- テスト方法: 異なる ユーザーアカウントでログインして Data Agent に同じ質問を投げる
```

#### セキュリティチェックリスト

- [ ] Data Agent は認証済みユーザーのみがアクセスできる
- [ ] テナント外への情報流出がないことを確認
- [ ] 機密データ（作業者個人情報）へのアクセスが適切に制御されている
- [ ] 管理者がアクセスログを確認できる（Fabric 監査ログ）
